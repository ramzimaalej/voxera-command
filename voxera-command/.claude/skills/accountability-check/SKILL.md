# accountability-check

Surface what's slipping. Pairs with `priorities-coach` (which looks forward) — this skill looks backward.

A good coach notices when "close customer #4" has been on your top-3 for 4 weeks running but hasn't moved. The skill does that without judgment but without softness either: it surfaces the pattern, asks pointed questions, and forces a decision (kill, downgrade, or actually commit).

## When to invoke

- User asks "what's slipping" / "what have I been dropping" / "what's chronic"
- Before setting next-week priorities (so you see the carry-over picture clearly)
- Invoked by `weekly-business-review.js` as part of the coach-pushback task
- Monthly retrospective (look back 4+ weeks)
- After a long absence (post-vacation re-orientation)

## Inputs

- `weeksToReview` (default 4): how many weekly notes back to analyze
- `focus` (optional): "priorities", "customer-commitments", "follow-up-commitments", "all" (default)

## State to read

1. **Last N weekly notes** — `voxera-command/docs/operations/weekly-reviews/*.md`, sorted by date, most recent first.
2. **Customer pulse files** — `voxera-command/docs/operations/customers/*.md`, particularly the `## Pulse log` and `## Open action items` sections.
3. **ADRs created/updated in the period** — to see if a slip was actually a strategy change.
4. **FEATs status transitions** — `voxera-command/docs/product/features/FEAT-*.md` — to confirm whether shipped work matches priorities claimed.
5. **BUGs status transitions** — same for bugs.

## Pattern detection

### 1. Chronic slippers — top-3 priorities repeated across weeks

For each weekly note's "Next-week priorities (top 3)" section, extract the items. Then for each item, search subsequent weeks:
- Did it appear in "Last week's commitments — how each landed" as done/shipped/closed? → completed; no slip.
- Did it appear again in a later week's "Next-week priorities"? → carry-over. Count repetitions.
- Did it never appear again, with no completion record? → silently dropped.

**Flag**:
- **Chronic slipper**: 3+ repetitions without completion. Surface the item, the # of weeks, the original framing vs current framing (has it drifted?).
- **Silently dropped**: 1 appearance, never again, no completion record. Ask whether intentional.

### 2. Commitments to others — past-due

For each weekly note's "Commitments to others" section, extract items with due dates. For each:
- Is the due date past?
- Is the commitment marked closed in any later weekly note's section?
- If past-due + not closed → **flag**.

These are higher-stakes than internal priorities because they involve someone else's expectations. Surface them loudly.

### 3. Customer-pulse declines without action

For each customer pulse file, scan the `## Pulse log` for health transitions:
- Any transition green → yellow → red without a corresponding closed action item.
- Any entry with `health: yellow` or `red` more than 2 weeks ago with no later health-improvement entry AND no closed action items.
- Any customer with `updated:` more than 4 weeks ago (`health: unknown` → relationship is going cold).

### 4. Strategic drift

Cross-reference what was shipped in the period (closed FEATs/BUGs, new ADRs, customer pulse updates) against the strategy:
- Did the period's shipped work map to roadmap Now items? If not, why was it done?
- Were any motion-defining items NOT touched at all in the period? (e.g., zero English-markets work across 4 weeks despite ADR-0009 saying it's a major motion).

### 5. Review cadence itself

- Was a weekly note skipped in the past N weeks? List the missed dates.
- Was the latest weekly note generated by the process, or only manually? (Frontmatter / structural shape will tell.)

## Output

```markdown
## Accountability check — past <N> weeks (ending YYYY-MM-DD)

### Chronic slippers (3+ weeks)
- **"<item>"** — repeated <N> weeks (first appeared <date>). Current framing: <quote>. Status: not done.
  - **Kill, downgrade, or commit?** <pointed question forcing a decision>

### Silently dropped
- **"<item>"** (committed <date>, never seen again). Intentional or forgotten?

### Past-due commitments to others
- **<person>**: "<commitment>" was due <date>. Not marked closed. Reach out today.

### Customer-pulse signals not addressed
- **<customer>** went `yellow` <date>. Action items from that entry: <list>. None closed.
- **<customer>** last updated <date> (> 4 weeks). Health: unknown. Schedule a check-in.

### Strategic drift
- <observation>: e.g., "Shipped 3 FEATs in the period; 0 were in the roadmap Now column when committed to."
- <observation>: e.g., "ADR-0009 names English-markets as a major motion. 0 weekly notes in the period set an English-markets top-3 priority."

### Review cadence
- Weekly notes in the period: <count>. Skipped: <missed dates if any>.

### Pointed questions (pick the ones that sting most)
1. <question 1>
2. <question 2>
3. <question 3>
```

## Voice

Direct, specific, evidence-cited. Examples:

- ✅ "'Close customer #4' has been on your top-3 for 4 weeks. Either you don't know how to close them, or it's not actually a priority. Which?"
- ✅ "You committed to 'reach out to Marcus by 2026-05-17.' Today is 2026-05-31. Reach out today or kill the commitment."
- ✅ "Customer-3 went yellow 3 weeks ago — 'pricing concerns raised.' No follow-up action item closed since. The longer you wait, the more this looks like avoidance."
- ❌ "It looks like a few items have been carried over — totally normal!"

## When the picture is clean

If no slippage, no past-due commitments, no customer declines unhandled: say so plainly.

```markdown
## Accountability check — past 4 weeks (ending 2026-05-31)

Nothing chronic to surface. Weekly cadence held (4/4 reviews). No past-due commitments. All yellow customer pulses have closed action items. Strategic drift: none — all shipped work maps to roadmap Now.

Carry on. Use `priorities-coach` for next-week's set.
```

Don't pad — silence-when-clean is part of the trust.

## Don't

- **Don't flag every carry-over.** 1-2 weeks of carry-over is normal. 3+ is the chronic-slipper threshold.
- **Don't moralize.** Surface the pattern, ask the question, let the user decide. "Why have you been avoiding this?" is fine; "you're avoiding this because…" is not.
- **Don't fabricate.** If a weekly note doesn't have a "Commitments to others" section, don't invent one. If a customer pulse file has no entries in the period, say so.
- **Don't recommend new priorities** — that's `priorities-coach`. This skill is read-only on the backward-facing picture.
